## Сценарій інтеграції
**Назва:** Динамічний розрахунок поїздки TaxiLink
**Проблема користувача:** Водію потрібно бачити точну адресу (а не координати), актуальний час прибуття (з урахуванням погоди) та вартість, еквівалентну світовим валютам, для прийняття рішення про поїздку.

## Використані зовнішні API
| API | Endpoint | Отримані дані | Кешування |
|---|---|---|---|
| Open-Meteo | `/v1/forecast?...` | погодні умови (WMO код) | ні (потрібні live-дані) |
| Nominatim | `/reverse?lat=...&lon=...` | текстова адреса за координатами | ні |
| OpenRouteService | `/v2/directions/driving-car` | точна відстань та базовий час | ні |
| NBU API | `/NBUStatService/v1/statdirectory` | актуальний курс USD | 12 год (IMemoryCache) |

## Fallback-стратегії (Polly реалізовано в Program.cs)
- **Open-Meteo недоступний** → сервіс повертає коефіцієнт 1.0 (стандартні умови), націнка не застосовується, застосунок продовжує працювати.
- **Nominatim недоступний** → на UI підставляються сирі координати замість назви вулиці.
- **NBU API недоступний** → береться останнє значення з кешу; якщо кеш порожній — сума в доларах не відображається або рахується за дефолтним статичним курсом.

## Власний Web API
- Controller: `OrdersApiController` (Головна сутність - Order)
- Swagger: `/swagger`
- Тест: `GET /api/orders` - отримує всі замовлення системи.

## Як запустити
1. Переконатися, що налаштовано рядок підключення до БД в `appsettings.json`.
2. Додати API ключ для OpenRouteService в `appsettings.Development.json`
3. Виконати `dotnet run`.
